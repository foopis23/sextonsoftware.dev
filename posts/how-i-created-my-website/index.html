If you have some development skills, and are looking to create a blog site for free, this is the place for you. I am going to walk through the steps I took to create my blog site. We are going to create this blog site using the static site generator Hugo and then host it on github pages.

I know this site is mostly to document my game developer journey, but I think this is a good thing to know for game developers and plus, I learned so I want to share it.

0. Prerequisites

  1. A Github account and basic experience using git.
  2. An understanding of how to add a program to path on your operating system or knowledge on how to use your OS’s package manager.
  3. Some kind of code or text editor software.

1. Install Hugo

There are a few methods for installing Hugo on your machine. Probably the best solution is using homebrew, as it requires the least setup. If you do not have homebrew, then you can download a binary of Hugo from github releases and add the binary to your system path. See the link down below for more details.

2. Create Hugo Project

Next we need to create a new project. Open your command line and navigate to the directory you want to create the project directory in. Then run this command. Make sure to replace <project-name> with whatever you want to call your project.

hugo new site <project-name>

This command will create a new directory in the current directory. To enter the project from the command line, run

cd <project-name>

Now that you are in the project directory, run the following command to initialize this project as a git project.

git init

Finally, add a file named .gitignore to the project directory. This file will be used to ignore files that you do not want to include in the project. Paste the following text into the file.

# Generated files by hugo
/public/
/resources/_gen/
/assets/jsconfig.json
hugo_stats.json

# Executable may be added to repository
hugo.exe
hugo.darwin
hugo.linux

# Temporary lock file while building
/.hugo_build.lock

3. Adding a theme

Now we need to add a theme to the site. If you don’t plan to modify the src code for the theme to much, the best option is to add it as a git submodule (that way you can get updates for the theme down the line with one command). A lot of themes will have commands to add the theme as a submodule. There is a list of themes on Hugo's websites . For my website, I am using the hello-friend-ng theme by Djordje Atlialp . I am going to walk through how I added that to my site.

Once in the project directory, run the following command to add the theme as a submodule.

git submodule add https://github.com/rhazdon/hugo-theme-hello-friend-ng.git themes/hello-friend-ng

You will find a config.toml in your project directory. Open that in your preferred text editor and paste the following defaults for the theme.

baseurl      = "localhost"
title        = "My Blog"
languageCode = "en-us"
theme        = "hello-friend-ng"
paginate     = 10

[params]
  dateform        = "Jan 2, 2006"
  dateformShort   = "Jan 2"
  dateformNum     = "2006-01-02"
  dateformNumTime = "2006-01-02 15:04"

  # Subtitle for home
  homeSubtitle = "A simple and beautiful blog"

  # Set disableReadOtherPosts to true in order to hide the links to other posts.
  disableReadOtherPosts = false

  # Enable sharing buttons, if you like
  enableSharingButtons = true

  # Metadata mostly used in document's head
  description = "My new homepage or blog"
  keywords = "homepage, blog"
  images = [""]

[taxonomies]
    category = "blog"
    tag      = "tags"
    series   = "series"

[languages]
  [languages.en]
    title = "Hello Friend NG"
    subtitle = "A simple theme for Hugo"
    keywords = ""
    copyright = '<a href="https://creativecommons.org/licenses/by-nc/4.0/" target="_blank" rel="noopener">CC BY-NC 4.0</a>'
    readOtherPosts = "Read other posts"

    [languages.en.params.logo]
      logoText = "hello friend ng"
      logoHomeLink = "/"
    # or
    #
    # path = "/img/your-example-logo.svg"
    # alt = "Your example logo alt text"

  # And you can even create generic menu
  [[menu.main]]
    identifier = "blog"
    name       = "Blog"
    url        = "/posts"

If you are using a different theme, follow the setup instructions for that theme.

4. Adding Content

Finally, the part we have been waiting for, we get to create content. To create new content with all the necessary meta tags we will use the hugo new command. To create a new blog post. Run the following command.

hugo new posts/my-first-post.md

Now if you open that new content file in a text editor, located at <project-dir>/content/posts/my-first-post.md, you will see something like this.

---
title: "My First Blog Post"
date: 2022-07-04T09:24:14-04:00
draft: true
toc: false
images:
tags:
  - untagged
---

Don’t worry if you don’t have all of this, some themes add different meta tags to posts. The important part is that at the top of your file you have a block of text surrounded with ---. Right under this block you can just start typing. All of the content is made in markdown while allows you to do simple styling. Here is a Markdown Cheatsheet get you starter.

An example of a page might look something like this.

---
title: "My First Blog Post"
date: 2022-07-04T09:24:14-04:00
draft: true
toc: false
images:
tags:
  - untagged
---

Hello this is my first blog posts. I just wanted to say hi.

## Who Am I?
Im the guy who created this blog.

5. Start the Hugo Server

Now that we have our blog site, we need to start the server. To do this, we will run the following command.

hugo server -D

Hugo server starts a server running on our local machine. The -D flag will include draft content in the site for preview. In your command line, after running that command, you should see a line that looks like this:

Web Server is available at http://localhost:1313/ (bind address 127.0.0.1)

This means that you can go to http://localhost:1313/ in your browser to see your site. You can make changes to the content and see the changes immediately.

6. Create a Repository on Github

Now that we have a site, we need to create a repository on Github. This is where you will be able to share your site with the world. To create a new repo on github, you will need an account at https://github.com. Once you are signed in you will be able to click the plus icon in the top right corner of the screen and select New Repository. From there, you you can name your repo and select the Public option. Leave everything else default or blank.

Now we need to push or local project to the github repository. To do this, we will run the following commands in the project directory.

git add .
git commit -m "Initial commit"
git remote add origin git@github.com:<username>/<repo-name>.git
git branch -M main
git push -u origin main

This command will stage all of the changed files, commit them to the default branch, add a remote origin for the project, set default branch’s name to main and push the main branch to the github repository.

7. Deploy to Github Pages

Now that we have a repository on Github, we need to deploy it to Github Pages. To do this we need to setup a github actions workflow. Don’t worry if you don’t know exactly what this is. It is just a way to to automatic build and deploy our website to github pages when we push our content to github. To setup a workflow up, create a new directory called .github in the project directory. Then in the .github directory, create a new directory called workflows. Then, create a new workflow file called main.yml. In that file, paste the following.

name: CI
on:
  push:
    branches:
      - main
jobs:
  deploy:
    runs-on: ubuntu-18.04
    steps:
      - name: Git checkout
        uses: actions/checkout@v2

      - name: Update theme
        # (Optional)If you have the theme added as submodule, you can pull it and use the most updated version
        run: git submodule update --init --recursive

      - name: Setup hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: "0.101.0"
          extended: true

      - name: Build
        # remove --minify tag if you do not need it
        # docs: https://gohugo.io/hugo-pipes/minification/
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
          publish_branch: gh-pages

What this action does is, on push to the main branch, checkout the code, pull the theme, install hugo in the action container, build the project, and deploy the built site to a new branch called gh-pages.

Next you need to set the base url for your site. This is the url that will be used to access your site. If you are not going to buy a domain and set it up for this, then the baseurl will be https://<username>.github.io/<repo-name>. If you do use a custom domain, then you can set the baseurl to https://<domain-name>. To set the base url, open the config.toml file in the project directory and set the baseurl to the domain you will be using.

After this is added, you can run the following command to push your code to github.

git add .
git commit -m "Added Automatic Deployment"
git push

8. Setup Github Pages on Repository

The last thing we need to do is to setup Github Pages on our repository. To do this, we will need to go the repository settings page. You can find this on your Github repository page by clicking the gear icon in the nav bar. Its the button farthest to the right. Once in the settings page, you can click the Pages tab in the left side nav. From there, under “Source” section, you can select the gh-pages branch and leave the folder set to /(root).

9. The Future / References

You can now see your site in Github Pages. To add more content to your site, follow the steps above for adding content and then to deploy it to Github Pages. Run the following commands.

git add .
git commit -m "Content Update Message/Label"
git push

Also if its easier for you to edit this files in the browser, you can press . while on the github repository page and you will be able to edit files there.
